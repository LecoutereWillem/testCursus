BRANCHES

toon alle branches in git:
git branch
nieuwe dev-branch aanmaken:
git branch dev
kan ook met checkout in 1 beweging:
git checkout -b dev //automatisch overschakelen naar nieuwe dev
gewoon overschakelen via checkout:
git checkout branchnaam
nieuwe branch syncen met remote:
git push -u origin nieuweBranch
Samenvoegen van branches:
git merge branchnaam //voegt aangegeven branch toe aan huidige branch
Er kan ook gemerged worden van feature-branch into the dev-branche:
git merge dev//terwijl actieve branch = feature-branch verwijderen van een branch:
git branch -d branchnaam //lokaal verwijderen
git push origin --delete branchnaam //remote branch verwijderen

CONFLICTEN

Indien geen conflicten en branch is 1 x afgetakt en er is niets veranderd aan hoofdtak:
FAST FORWARD
Indien complexere branch-structuur, de ene heeft commit die de andere niet heeft
3 WAY MERGE
methode:
1. Eerst dev mergen in feature1 branch
m 2. files aanpassen
3. merge dev in feature2 branch
4. files aanpassen
5. merge feature1 into dev
6. merge feature2 into dev

de optie --no-ff
git merge --no-ff featurebranch
Er wordt nu wel een commit boodschap meegegeven, bij FF niet.

Indien wel conflict: ofwel lokaal oplossen, ofwel rechtstreeks op github.

  1. lokaal oplossen
    vanuit de feature-branch, dev mergen= git merge dev
    conflict: accept current of incoming change = save = >
    conflict opgelost : add => commit => push
  2. via github online
    Resolve Conflict bij Pull-Request dus ook hier : Dev binnentrekken op feature branch, daar conflict oplossen, en commit merge.
    Vanuit Github lokaal binnen trekken?
    git pull

GIT FIXES : terugkeren in geschiedenis op 2 manieren:

working tree: default branch.
clean working tree: situatie root map == commit, geen losse bestanden, alles gecommit.
check: git status
untracked files : zijn nog niet geadd om meegenomen te worden in commit.
niet gecommitte wijzigen ongedaan maken:
git reset //unstage alle gestagede wijzigen
git checkout . //voert checkout uit naar vorige commit
gewijzigde bestanden worden hersteld, maar toegevoegde bestanden blijven wel staan. Wil je deze weg?
git clean --force //een clean working tree is het resultaat

1. Wissen van commits via reset (specifiek per branch).//gezien als rollback Kan voor veel problemen zorgen indien er ook remote wordt gewerkt en er gepushed wordt.
git reset commitnummerKort //project wordt hersteld naar bepaaLde commit
Indien problemen met remote:
git push --force // !!!!!! oppassen: commits verdwijnen op remote. 2. Nieuwe commit die de situatie aanpast aan meegeven oude commit via revert. De commit history blijft behouden, daarom belangrijk om veel kleine commits te doen, maakt foutopsporing makkelijk.
git revert commitnummer//een nieuwe commit is aangemaakt
